1、什么是虚拟DOM

虚拟DOM是真实DOM在内存中的表示

2、函数组件与类组件区别

函数组件：没有生命周期、this、state

类组件：有生命周期、this、state

函数组件的性能比类组件的性能要高，因为类组件使用的时候要实例化，而函数组件直接执行函数取返回结果即可。为了提高性能，尽量使用函数组件。

3、props与state的区别

props与state都是普通的js对象，都包含影响渲染输出的信息，但是在组件方面的功能是不同的

state是自己管理数据，控制自己状态，可变

props书外部传入数据参数，不可变

有state的叫做有状态组件

多用 props，少用 state，也就是多写无状态组件

4、什么是高阶组件

高阶组件（HOC）接受一个组件并且返回一个新组件的函数，也叫纯组件

HOC用例：

  • 代码重用、逻辑、和引导抽象
  • 渲染劫持
  • state抽象和操作
  • props处理

5、什么是JSX

JSX是将原始HTML模板嵌入到JS代码中。JSX代码本身不能被浏览器解析，要通过Babel和webpack等工具转换为传统JS

6、react生命周期

挂载

当组件实例被创建并插入 DOM 中时，其生命周期调用顺序如下：

  • constructor()
  • static getDerivedStateFromProps()
  • render()
  • componentDidMount()

更新

当组件的 props 或 state 发生变化时会触发更新。组件更新的生命周期调用顺序如下：

  • static getDerivedStateFromProps()
  • shouldComponentUpdate()
  • render()
  • getSnapshotBeforeUpdate()
  • componentDidUpdate()

卸载

当组件从 DOM 中移除时会调用如下方法：

  • componentWillUnmount()

错误处理

当渲染过程，生命周期，或子组件的构造函数中抛出错误时，会调用如下方法：

  • static getDerivedStateFromError()
  • componentDidCatch()

7、Hooks

Hooks 通常支持提取和重用跨多个组件通用的有状态逻辑，而无需承担高阶组件或渲染 props 的负担。Hooks 可以轻松地操作函数组件的状态。

Hooks 在类中不起作用，通过使用它们，咱们可以完全避免使用生命周期方法。

8、Virtual DOM工作原理

Virtual DOM 是一个轻量级的 JavaScript 对象，它最初只是 real DOM 的副本。

  • 每当底层的数据发生改变时，将Virtual DOM 重新渲染
  • 计算之前DOM与新的DOM之间差异
  • 完成计算后，将只有实际更改的内容更新 real DOM

9、什么是Redux

它是 JavaScript 程序的可预测状态容器，用于整个应用的状态管理。使用 Redux 开发的应用易于测试，可以在不同环境中运行，并显示一致的行为。

三个原则

  • ​ 单一数据来源
  • ​ 状态是只读的
  • ​ 使用纯函数进行修改

四大组件

store：用来存储数据
reducer：真正的来管理数据
actionCreators：创建action，交由reducer处理
view： 用来使用数据，在这里，一般用react组件来充当

1.store通过reducer创建了初始状态
2.view通过store.getState()获取到了store中保存的state挂载在了自己的状态上
3.用户产生了操作，调用了actions 的方法
4.actions的方法被调用，创建了带有标示性信息的action
5.actions内部通过调用store.dispatch方法将标志性的action发送到了reducer中
6.reducer接收到action并根据标识信息判断之后返回了新的state
7.store的state被reducer更改为新state的时候，store.subscribe方法里的回调函数会执行，此时就可以通知view去重新获取state
